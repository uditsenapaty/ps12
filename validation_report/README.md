# validation_report/ — committed validation results

Validation of the interpolation models against **real higher-cadence ground truth** (GOES-19 /
Himawari, 10-min). These files are **committed** (heavy raw `.nc`/weights stay gitignored) and are
rendered in the dashboard's **Validation Report** tab.

## Validation protocol (matches the PS)
From a 10-min source we take two frames **20 minutes apart** (e.g. 00:00 and 00:20) as input, predict
the **held-out real 00:10 middle frame**, and score the prediction against that ground truth. This is
done across many triplets per model. (INSAT uses the analogous **leave-one-out** at 30 min: predict the
real 00:30 from 00:00 & 01:00.)

## Each experiment subfolder (e.g. `validation_report/goes/`, `.../himawari/`, `.../insat/`)
- `report.md` — per-model validation table (PSNR, SSIM, FSIM, edge-SSIM, MSE, MAE-K, LPIPS)
- `metrics_raw.json` — per-triplet raw metrics (auditable)
- `psnr_by_model.png`, `ssim_by_model.png` — bar charts
- `comparison_triplet0.png` — qualitative panel: inputs, GT, each model's prediction + error maps
- `timelapse_<model>.gif` — original-vs-interpolated time-lapse

Generated by `src/eval/report.run_eval(...)` (see `walkthrough.md` Step 8) on the GPU server, then
committed so the validation is reproducible and visible without re-running the GPU job.
